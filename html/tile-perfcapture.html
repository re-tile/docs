Name

tile-perfcapture — converts perf events data files into XML format used by Profile UI

Synopsis

tile-perfcapture [-h | --help] [-c | --cd directory] [-o | --output profile.xml] [-e | --events profile_events.xml] [-l | --vmlinux /path/to/vmlinux] [-v | --verbose] [-p | --show-progress] [-S | --show-samples] [-P | --pretty-print] [perf.data ...]

Description

The tile-perfcapture tool is used to convert the raw profile statistic data files output by the perf command into an XML file that can be viewed in the tile-profileui viewer.

Note: in normal usage, this command is invoked by the tile-profileui itself as part of doing a profiling run. Users should normally not need to invoke this command directly.

This tool is needed because the profile data generated by the perf utility is in an internal format and hence pretty much inaccessible to tools other than those included with it. The XML format produced by tile-perfcapture is more regular, self-contained, self-documenting, and easy to process.

Command-Line Options

-h | --help

Optional. Displays usage information and list of command options, then exits.

-c | --cd directory

Optional. Sets the default directory for subsequent pathname options. Can appear more than once. Before the first instance of this option, the default directory is the directory in which tile-perfcapture was invoked.

-o | --output profile.xml

Optional. Specifies the output pathname of the resulting profile.xml file. If not specified, defaults to profile.xml in the current directory.

-e | --events profile_events.xml

Optional. Specifies an XML file descripting the profiling events available on the hardware platform. If not specified, defaults to profile_events.xml file in the TILE file system.

-l | --vmlinux /path/to/vmlinux

Optional. Specifies vmlinux binary to use for OS/kernel symbol information. If not specified, defaults to /boot/vmlinux. If the vmlinux file is not found or does not contain symbol information, the profile data can still be processed, but there will be more unresolved OS/kernel addresses in the profile.xml output.

-v | --verbose

Optional. Displays debugging information. By default, this command operates silently except for usage/execution error messages.

-p | --show-progress

Optional. Displays progress output messages and other useful information.

-S | --show-samples

Optional. Displays details of sample data read from sample file.

-P | --pretty-print

Optional. Pretty-prints XML output rather than using normal "flat" format. Makes XML more human-readable at cost of increased file size.

perf.data

A perf.data file to process. (See description below under Usage.)

Usage

When you run the perf record command, it generates a raw profile data file (by default named perf.data). One can then run the tile-perfcapture command on this output file to produce a profile.xml file that is readable by the Profile UI tool.

The resulting profile.xml file is self-contained, and can be copied/archived wherever you like. The original sample capture directory can then be deleted if you no longer need it and want to reclaim the space.

Note: in normal usage, this command is invoked by the tile-profileui itself as part of doing a profiling run. Users should normally not need to invoke this command directly.

Also, to help keep things simple, the tile-profileui tool is currently only built for and only run on TILE, so we only discuss running it on TILE below.

Using tile-perfcapture on TILE

You can run the native tile-perfcapture command on a perf.data file to produce a profile.xml file: 

    perf record -g -c 1000000 \
      -e ONE,INSTRUCTION_STALL,INSTRUCTION_BUNDLE,LOAD_STALL \
      -o perf.data \
      -- \
      your_app arg1 ... argn

    tile-perfcapture -p -o profile.xml perf.data

Yyou can then view the resulting profile.xml file in the Profile UI by starting the Profile UI, entering the profile.xml pathname in the "Capture To:" pathname text box, then clicking any of the view buttons to display a view based on data in the file.

Metadata files needed by tile-perfcapture

The tile-perfcapture command requires an profile_events.xml metadata file in order to properly process your profiling data. If you're running tile-perfcapture on TILE, it will automatically locate this file in the MDE file system. You can optionally override the default location for this file, but you should rarely have to do this.

The profile_events.xml file provides Tilera-specific metadata about the hardware events for the TILE platform. This file can be found in your MDE installation: 

    ${TILERA_ROOT}/etc/info/chip/10/profile_events.xml

or 

    ${TILERA_ROOT}/tile/usr/share/tilera/info/chip/10/profile_events.xml

Note: If you specify the events file explicitly, make sure that you use the correct events metadata file for your architecture; in other words, use the metadata file from a Gx MDE installation when profiling on Gx hardware.
