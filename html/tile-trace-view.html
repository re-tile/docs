Name

tile-trace-view — View Tile Processor traces

Synopsis

tile-trace-view [ file ...]

Description

tile-trace-view is a graphical viewer for traces produced by tile-sim.

Generating Trace Information

To generate traces, add the --trace-tile-view prefix to tile-sim. This will produce output files with the following names:

File Types

prefix.xml

The aggregated statistics file.

prefix.tile*.router

The network trace files.

prefix.tile*.mainproc

The tile processor trace files.

One can also generate "interval profile" files using tile-sim's --profile-interval interval option along with the --xml-stats or --xml-profile options. In this way, a separate XML profile file is generated per interval, which enables much longer runs to be traced (at a coarser granularity). Interval profiles can be displayed in the Trace Chart by loading the prefix.number.xml files.

Running tile-trace-view

tile-trace-view is a Java program, and so will run on both Linux and Windows.

In the Linux distribution, a launcher script named tile-trace-view is available in the "bin" directory.

On Windows, you should create an empty directory and unzip etc/tile-trace-view-win32.zip into it; this will create a launcher batch file named tile-trace-view.bat.

When you run tile-trace-view, you can pass as arguments the list of trace files you want the program to process and summarize. So, for example: 

tile-monitor --simulator ... --sim-args -+- --trace-tile-view ttv -+- ...

tile-trace-view ttv.*

You can also select the list of trace files from within the trace viewer:

  • Run tile-trace-view without arguments.

  • Click "File" in the menu bar, then "Open".

  • Use the file-browser to navigate to the directory containing the trace files.

  • Multi-select (e.g. click the first and SHIFT-click the last) the set of trace files (*.xml, *.router, and *.mainproc).

  • Click "OK".

Using the Trace Viewer

Three views or visualizations are provided, as follows:

Trace Chart

The graph's horizontal axis is time, in cycles; the vertical axis is the tile number. For each tile, events are displayed in horizontal "bands", a broad one on top for the processor, and a narrower one on bottom for the selected network.

Each band is a stacked bar-chart, with colors corresponding to time spent in various "events" (for instance processor execution, stalls of various types, and network traffic destinations). The events are color-coded according to the legend on the right.

Events displayed in the legend are selectable: one can turn on and off individual events (for instance to de-clutter the display or pick out the contribution of specific events), and one can change the color of an event type by clicking its button in the legend.

To get more information about an event, hover the cursor over it. To make it easier to select a specific event, zoom in.

The two ways to zoom in are to use the X and Y Zoom controls, or to "rubber band". To do the latter, select Zoom In in the Mouse Drag box, left-click in the graph, and then while holding the left mouse button down, drag the corner of the bounding box. Release to zoom. The Zoom Stack acts like browser backwards and forwards buttons, so one can revert to a previous zoom state, and then return back and so on.

When zoomed in, one can scroll in the X and Y dimensions, either by using the scroll bars, or selecting the Scroll option in the Mouse Drag box. The latter acts as a "grab scroll", moving the chart view along with the mouse. Grab scroll is also available via control-left-click regardless of the Mouse Drag selection.

Note: if you accidentally zoom in too far, i.e. by clicking on the graph and selecting a very small region, just click the left arrow in the Zoom Stack controls to return to the previous zoom size.

A number of different views are provided. (Note that views can only display data which is available, depending on the trace or profile files loaded, and the data available in them.)

Summary Activities

In the default view, detailed events are summarized into higher-level categories.

Detailed Activities

Detailed events are broken out individually.

Function Calls

Time slices are colored according to their location in the program, rather than the event type.

Application Events

Application-level events are charted, as derived from BEGIN/END macros placed in the application source. (In this view, the Network View options below are not available.)

Message Traffic

Show network traffic events, namely words being sent from a network port.

Input Queue Lengths

Show the length of the network input queues, with a separate histogram per source (the tile processor and the four cardinal directions North, East, West, and South).

Scatter Plot

As its name suggests, this visualization shows events distributed over time (cycles on the horizontal or X axis) and location (instruction or data addresses on the vertical or Y axis). One main processor trace at a time is selected in the right-hand-side legend.

Five variants are provided: all instruction or data addresses, and three special modes focused on cache effects in the level-two unified cache or level-one instruction or data caches. In the cache variants, the coloring is done according to the virtual address modulo the cache associativity set size rather than the event type. This makes it easier to pick out potential cache conflicts, which typically look like alternating cache misses at close-by vertical positions with different colors, which implies objects at different addresses mapping to the same cache lines.

Zooming, scrolling, etc. work as in the Trace Chart.

Network Animation

This visualization shows the state of the entire chip over time. In each tile, the central square represents the processor, and is colored according to event (executing instructions, stalls, etc.) The lower-right-hand area represents the router for the selected network, with five input queues for each of the cardinal directions and the processor itself. Network packets are indicated by arrows colored according to the destination tile. Hovering the mouse over a tile gives more detailed information on what's going on in the current cycle.

To scroll, use the controls on the bottom edge. On the left, use the buttons to scroll backwards and forwards a single cycle at a time. On the right, use the scrollbar for more global motion. (Tip: the scrollbar buttons will auto-repeat, giving a quick-and-dirty movie.)

Navigation between views

A mouse right-click will display a context menu, including Zoom choices and a Select View dialog, which can be used to navigate between views. The dialog is initially populated with a cycle and trace derived from the right-click location. These can be edited, and used to select a different view focused on the chosen cycle and trace.
