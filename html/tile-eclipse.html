Name

tile-eclipse — Eclipse-based Integrated Development Environment (IDE) for Tile Processor programming

Synopsis

tile-eclipse [--help] [--here] [--force] [--profile xml-file] [--jdb] [--max-memory size] [--locking { default | maximum | minimum | none } ]

Description

tile-eclipse is a visual Integrated Development Environment (IDE), based on the open-source Eclipse project.

The first time you start tile-eclipse, it will automatically select three Eclipse "perspectives", or sets of windows, that are customized for developing Tilera software: the "Author", "Debug", and "Profile" perspectives. Initially, the "Author" perspective is selected.

By default, tile-eclipse uses the path ~/.tilera/workspace as the Eclipse workspace directory. If you want to use a different path, set the environment variable TILERA_WORKSPACE to the desired path of the workspace directory.

You can also use the --here option to change the workspace directory to the current working directory for a single launch of tile-eclipse.

tile-eclipse includes integrated online help documentation. To view the online help, select "Help->Help Contents" from the Eclipse menu bar. When the help window appears, click the topic "Tilera MDE User Guide" in the left-hand "Contents" view. This will display the top level contents page of the user guide, from which you can navigate to the available documentation.

Please view the "Getting Started" topic before you begin using the GUI, since it covers such basic tasks as:

  • Creating a Tilera software project

  • Running a Tilera program (simulator or hardware)

  • Debugging a Tilera program (simulator or hardware)

  • Profiling a Tilera program (simulator or hardware)

If you have not used the Eclipse development environment before, you may wish to review an introductory user's guide, such as "Eclipse Distilled", by David Carlson, or the online documentation at the Eclipse web site or the online documentation in Eclipse itself.

Command-Line Options

--help

Display usage information.

--here

Use current working directory as workspace, instead of the default (~/.tilera/workspace). Overrides value of TILERA_WORKSPACE environment variable, if set.

--force

Remove the lock, if any, on the workspace left by a previous instance of tile-eclipse that was not shut down cleanly. Only use the --force option if absolutely sure there are no other instances of tile-eclipse running. Running multiple instances of tile-eclipse can corrupt the workspace.

--profile xml-file

Selects the Profile View and loads the specified profile XML file.

--jdb

Configures the JVM used by tile-eclipse to support debugging with jdb.

--max-memory size

Specifies the maximum heap size used by tile-eclipse. The default is 512m (m=megabytes).

--locking { default | maximum | minimum | default }

Specifies the workspace lock method, which can be default (same as maximum), minimum, or none. (Note: disabling workspace locking is not recommended; use --force instead.)
