Name

mpipe-stat — TILE-Gx mPIPE I/O Statistics

Synopsis

mpipe-stat [ { -i | --interval } interval ] [ -p | --preamble ] [ -r | --rates ] [ -t | --totals ] [ -v | --verbose ] [ -d | --packet-drops ] [ -h | --human-readable ] [ --config-lbl=drop | --config-lbl=bkt | --config-lbl=nr ] [ --config-ipkt=trunc | --config-ipkt=drop ] link1 [ link2 | ... | linkN ]

Description

mpipe-stat prints network I/O statistics, such as bytes and packets sent, received, and dropped. Note that it is a native tool, and must be run on the Tile Processor itself.

The link must be one or multiple on-chip mPIPE network interfaces: gbe1, gbe2, xgbe1, or xgbe2 etc.

For example: 

# /usr/sbin/mpipe-stat xgbe1 xgbe2
LINK     Tx Pkt       Tx bits    Rx pkt       Rx bits  Tx err    Rx err
--------------------------------------------------------------------------
xgbe1    635788      19934271     88832       9938956       0         0
xgbe2    243578       1683427      4423         68837       0         0

The LINK column shows the link names. The Tx Pkt column shows the total number of egress packets. The Tx bits column shows the total number of egress bits. The Rx pkt column shows the total number of ingress packets. The Rx bits column shows the total number of ingress bits. The Tx err column shows the total number of egress packets with errors. The Rx err column shows the total number of ingress packets with errors (i.e. CRC etc).

Options

-i | --interval interval

By default, mpipe-stat prints out statistics once. If you set an interval, mpipe-stat repeats the statistics every specified number of seconds.

-p | --preamble

Include the packet preamble and inter-frame gap when calculating bits per second and total bytes values.

-r | --rates

Print statistics per second instead of counts. Note that the --rates option requires that you set an interval using the --interval option. For example: 

# /usr/sbin/mpipe-stat xgbe1 xgbe2 --interval 10 --rates

LINK    Tx Pkt/s    Tx bits/s  Rx pkt/s     Rx bits/s Tx err/s  Rx err/s
--------------------------------------------------------------------------
xgbe1      35788     1934271      18832        938956       0         0
xgbe2      43578      163427       4423          6837       0         0
xgbe1      37781     1734712      17321        949568       0         0
xgbe2      45783      144277       4532          6773       0         0
...
-t | --totals

Print total accumulated statistics instead of counts per interval.

-d | --packet-drops

Print packet drop count or rate of an mPIPE instance per interval. 

# /usr/sbin/mpipe-stat xgbe1 xgbe2 --interval 10 --rates --packet-drops

LINK    Tx Pkt/s    Tx bits/s  Rx pkt/s     Rx bits/s Tx err/s  Rx err/s
--------------------------------------------------------------------------
xgbe1      35788     1934271      18832        938956       0         0
xgbe2      43578      163427       4423          6837       0         0
                                                      drop/0:     12.00/s
xgbe1      37781     1734712      17321        949568       0         0
xgbe2      45783      144277       4532          6773       0         0
                                                      drop/0:      7.00/s
...

drop/0: 12/s indicates mPIPE0 has 12 packets dropped per second in the 10 seconds interval.

Note: Packets dropped are per mPIPE instance and could be from multiple links.

-h | --human-readable

Print statistics numbers as human readable format. i.e. K: Kilo (1000), M: Mega (10^6), G: Giga (10^9), T: Tera (10^12) and P: Peta (10^15). 

# /usr/sbin/mpipe-stat xgbe1 xgbe2 --interval 10 --rates --packet-drops
--human-readable
LINK    Tx Pkt/s    Tx bits/s  Rx pkt/s     Rx bits/s Tx err/s  Rx err/s
--------------------------------------------------------------------------
xgbe1     35.78K        1.93M    18.83K       938.95K       0         0
xgbe2     43.57K      163.42K     4.42K         6.83K       0         0
                                                      drop/0:     12.00/s
...

-l | --list

Enumerate all link names.

--scan-drop

Cycle through packet drop reasons, one per given interval, then display each counter value, and repeat.

--config-lbl=drop

Reconfigure the Load Balancer statistics counter to count packets dropped by load balancer only (excluding those packets dropped by Classifier). The --config-lbl=drop option requires a --verbose option.

--config-lbl=bkt

Reconfigure Load Balancer statistics counter to count packets dropped by load balancer due to bucket full. The --config-lbl=btk option requires a --verbose option.

--config-lbl=nr

Reconfigure Load Balancer statistics counter to count packets dropped by load balancer due to NotifRing full. The --config-lbl=nr option requires a --verbose option.

--config-ipkt=trunc

Reconfigure iPKT statistics counter to count packets truncated due to out of iPkt buffer. The --config-iptk=trunc option requires a --verbose option.

--config-ipkt=drop

Reconfigure iPKT statistics counter to count packets dropped due to out of iPkt buffers. The --config-ipkt=drop option requires a --verbose option.

Notes

The rates statistics are calculated based on the time measured by the system's core reference clock, which is unlikely to be completely accurate; typical oscillators used for this purpose might vary by as much as 300 parts per million (0.03%). Ethernet standards also allow variance of up to 100 ppm (0.01%) in the output clock rate of the interface. This means that there may be some minor variation in the statistics reported on different boards, or when connected to different devices. For instance, a 10 Gbps interface which was in fact completely saturated might be reported to be running at anywhere from 9.996 Gbps to 10.004 Gbps.

When using --interval, or when running mpipe-stat several times in succession, you may notice that some values seem to reset from zero, while others do not.

  • Some values are incremental; each time they're displayed, you see the count of events since the last time the value was displayed. This is true of all of the MAC statistics (Tx/Rx packets, Tx/Rx bits, Tx/Rx packets of specific sizes, various types of error frames), This is because these values are maintained by hardware in clear-on-read registers; whenever mpipe-stat reads them, they're reset to zero.

  • Other values are cumulative; each time they're displayed, you see the total count of events since some time in the past. This is true of mPIPE Overall Statistics (cumulative) with --verbose. This is because these values are maintained by software; they are implemented as cumulative counters since that behavior is more useful. Use the --interval option to count these values incrementally during that execution of mpipe-stat.
