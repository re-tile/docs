Name

tile-mkboot — Boot Image Creation

Synopsis

tile-mkboot [ --hv-bin-dir dir ] [ -l | --linux ] [ --linux-dir dir ]
[ -n | --no-strip ] [ -c | --compress ] [ -f | --fixed ]
[ -v | --verbose ] { --hvc file.hvc ...} [ --hvd var[=value] ...] [ --hvi directory ...] [ --hvx value ...] [ --dump-hvc ] { -o | --output } outfile [outfile=]infile...

Description

tile-mkboot creates a stream of executable code and data which may be used to boot a Tile Processor. The boot stream consists of the booter and hypervisor code, followed by a hypervisor file system image containing files determined by the user. Normally those files consist of the hypervisor configuration file and the client operating system image.

Options

--hv-bin-dir dir

If the --hv-bin-dir option is specified, the hypervisor binary files (including the booters) will be taken from the directory dir; otherwise, they will be taken from ${TILERA_ROOT}/tile/boot when running on a host system, or from /boot when running natively on a Tile Processor.

-l , --linux

If the --linux option is specified, files necessary to boot a Linux kernel (currently just vmlinux) will be included in the output stream. In order for those files to be used, the hypervisor configuration file must specify vmlinux as a client binary name.

--linux-dir dir

If the --linux-dir option is specified, any files needed to boot a Linux kernel will be taken from the directory dir; otherwise, they will be taken from ${TILERA_ROOT}/tile/boot when running on a host system, or from /boot when running natively on a Tile Processor. This option is only useful with, and in fact implies, the --linux option.

-n , --no-strip

If the --no-strip option is specified, ELF executables specified as input files are copied unchanged into the output image. Normally, such files are first processed to have their symbol information removed, making the output image smaller.

-c , --compress

If the --compress option is specified, files listed in an input configuration file as a client supervisor will be compressed before inclusion in the output image, making the output image smaller. Only files which are also stripped are compressed, so --no-strip disables compression as well.

-f , --fixed

If the --fixed option is specified, tile-mkboot will not use the current hostname, or time and date, when generating comment fields within the output image. However, files used as input to the image generation process may still contain such data.

-v , --verbose

If the --verbose option is specified, extra informational output may be printed.

-o , --output outfile

Specify the name of the output boot stream file, unless --dump-hvc was provided, in which case it specifies the name of the output hypervisor configuration file.

--hvc file.hvc

Request the use of the given hypervisor configuration file.

More than one file may be specified, in which case their contents are concatenated.

--hvd var[=value]

Provide a variable definition to be used while processing hypervisor configuration files.

Inside hypervisor configuration files, such variables can be accessed using $var, and can be used with ifdef, ifndef, and defined(var).

Multiple --hvd values may be specified.

--hvi dir

Append a directory to the search path used to find hypervisor configuration files.

This search path affects both --hvc arguments and include directives inside hypervisor configuration files.

If multiple --hvi values are specified, they are searched in the order provided. Note that the system default directory for hypervisor configuration files is always added to the end of the list of directories to be searched; this directory is ${TILERA_ROOT}/tile/etc/hvc when running on a host system, or /etc/hvc when running natively on a Tile Processor.

--hvx value

Append a single space plus value to the definition of the special XARGS variable, which starts out defined, but empty. This variable is used by some hypervisor configuration files, usually to extend the primary client args.

--dump-hvc

Emit the result of preprocessing input hypervisor configuration files. If --output is specified, the result is written to that file. Otherwise it is printed to standard output.

[outfile=]infile

Specify a file to be included in the hypervisor filesystem image. If the outfile= prefix is included, the input file infile is added to the filesystem under the name outfile. Otherwise, the name in the output filesystem is identical to the input file name.

The hypervisor configuration specified by using one or more --hvc options becomes a file called "config" in the generated filesystem. This is subject to change, but the filename "config" should be considered reserved. The name of the client operating system is specified in the configuration file, but is conventionally vmlinux.

Example

In order to create a new boot stream myboot.bootrom with a customized hypervisor configuration file myconfig.hvc, and then boot it on a TILExpress card, one could do the following (but see below for a better technique): 

tile-mkboot --output myboot.bootrom --hvc myconfig.hvc --linux
tile-monitor --bootrom-file myboot.bootrom --pci

Note that tile-monitor will automatically run tile-mkboot if needed to generate a temporary bootrom file from which to boot. It passes through the --hvc, --hvd, --hvi, --hvx, and --hv-bin-dir options. It supplies an appropriate vmlinux=vmlinux argument, using ${TILERA_ROOT}/tile/boot/vmlinux by default. It allows the --bme and --mkboot-args options to be used to specify other extra arguments. And, it automatically provides various --hvd arguments appropriate to the target platform. So it is usually best to use tile-monitor instead of tile-mkboot. For example, the commands above could have been replaced by: 

tile-monitor --pci --hvc myconfig.hvc
