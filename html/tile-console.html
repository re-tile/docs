Name

tile-console — Interact with the Tile Processor console

Synopsis

tile-console [ --cu ] [ --dev DEV ] [ --serial SERIAL ]

Description

tile-console provides access to the console port on the Tile Processor, which displays Linux boot messages and normally provides an interactive Linux login, via a serial device on the host.

By default, tile-console uses the host tool kermit to communicate with a serial device. If the --cu option is used, then the host tool cu will be used instead. See the man pages for kermit and cu for more information about these host tools.

When using kermit, the special escape character is Ctrl+backslash ("^\"). After typing this character, various other characters will have special effects. Use "?" to see a list of these characters, "q" to quit, or "z" to suspend.

When using cu, the special escape character is tilde (~), at the start of a line. After typing this character, various other characters will have special effects. Use "?" to see a list of these characters, "." to quit, or "z" to suspend.

Note that ssh also uses tilde as a special escape character, so if you use ssh to connect to the machine with the serial port, and then use cu to connect to the serial device, you will need to use two tildes as the cu escape sequence.

The desired serial device can be specified directly via --serial. Otherwise, tile-dev will be used to pick a device, and the serial device will be taken from the serial=path line in the corresponding config file (/etc/tilepciN.conf, /etc/tileusbN.conf, or /etc/tilenet/ADDR.conf).

tile-console uses locking to ensure that only one program may connect to the same device at a time. The tile-monitor command uses the same locking (when appropriate).

Note that permissions must be set appropriately on the serial device and the lock directory. If not, tile-console may emit an error message with information about setting permissions.

Options

--cu

Use cu instead of kermit.

--dev DEV

Use tile-dev DEV instead of tile-dev to pick the device from whose config file the serial device will be acquired.

--bmc IP[:Node]

Use IP address and Node number to pick the serial device for tile-console. Default [:Node] number is 0.

--serial SERIAL

Use the serial device whose path is SERIAL.
