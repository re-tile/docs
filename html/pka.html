Name

pka — /dev/pka*/* special files for hardware acceleration of vector math operations used for public key authentication

Synopsis

#include <asm/pka.h>
#include <gxio/pka.h>

Description

The /dev/pka*/* special files provide user application access to the PKA acceleration hardware engines on the MiCA shims. There is one /dev/pkaX directory per MiCA Crypto shim, where X represents the shim number in the directory name. There is one special file for each command queue in the /dev/pka* directory.

Vector operations for public key authentication, such as modular exponentiation with Chinese remainder theorem for RSA authentication, can be offloaded from the core to the PKA engine via these special files. The ioctls described here provide a direct interface to the PKA engine's command queue interface.

Operations are placed onto the queue via the PKA_QUEUE_CMD ioctl, and the results of the command retrieved via the PKA_DEQUEUE_RES ioctl. The commands and the format of the operands are documented in the I/O Device Guide (UG404) Appendix C. The data structures used for these ioctls, pka_cmd_desc_t for commands and pka_res_desc_t for results, are defined in <gxio/pka.h>.

Configuration

By default, each pka device has 4 queues with 7 elements that are 2048 bytes each. The number of queues and the size of the element in each queue can be changed at module load time or at Linux boot time via the module parameters. The available memory is divided equally among the queues, and each queue is divided into as many elements as possible based on the specified element size. The elements in each queue must be sized so as to fit all operands for the type of command that will be submitted to that queue.

Queue elements should be sized such that all operands (and any necessary padding) for a command can fit in the space allotted to the element. See Appendix C of the I/O Device Guide (UG404) for information on the operand format for each of the supported commands.

Each pka major device is individually configurable via an array of module parameters. The array index corresponds to the shim number. The module parameters are:

num_hw_queues

The number of queues (1 - 4) to create. Available space is divided equally amongst the number of queues specified here. The default is 4.

q0_high_priority

If non-zero (true), queue 0 will receive high priority. The default is 0.

q0_elem_size

The number of bytes to be used for the data portion of each entry for queue 0. The default is 2048.

q1_elem_size

The number of bytes to be used for the data portion of each entry for queue 1. The default is 2048.

q2_elem_size

The number of bytes to be used for the data portion of each entry for queue 2. The default is 2048.

q3_elem_size

The number of bytes to be used for the data portion of each entry for queue 3. The default is 2048.

Example Configuration

When booting with tile-monitor, the module parameters can be passed to the Linux boot command line via the --hvx switch. For example, 

tile-monitor --hvx pka.num_hw_queues=1,2 --hvx pka.q0_elem_size=1024,512

will configure the pka engine on shim 0 to use only one queue with elements with 1024 bytes of space for operands, and the pka engine on shim 1 to use two queues (512-byte elements on queue 0 and 2048-byte elements on queue 1).

Ioctls

PKA_QUEUE_CMD

Place a command descriptor on the PKA engine's queue.

PKA_DEQUEUE_RES

Retrieve the result of a completed command from the PKA engine's queue.

Usage

The PKA_QUEUE_CMD ioctl is used to place a vector math command on the queue for processing by the PKA engine. A pointer to a pka_cmd_desc_t is passed as the argument to the ioctl.

The PKA_DEQUEUE_RES ioctl is used to retrieve the result of a vector math operation from the PKA engine's result queue. A pointer to a pka_res_desc_t is passed as the argument to the ioctl.

Errors

PKA_QUEUE_CMD errors:

EBUSY

The command queue is full.

EFAULT

Error placing command on the pka engine's queue.

There are certain restrictions on the values of the operands for some of the commands. They are listed in the I/O Device Guide (UG404) Appendix C. If these restrictions are not met, an incorrect result may be returned with no indication of an error.

PKA_DEQUEUE_RES errors:

EAGAIN

There are no completed results on the queue.

EFAULT

Error dequeuing result from queue.

See Also

See the I/O Device Guide (UG404) for more information on the PKA acceleration engine and commands.
