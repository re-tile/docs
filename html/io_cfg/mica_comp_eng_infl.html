    Performance Counter Control 0 (MICA_COMP_ENG_INFL_REG_PERF_CTL_0: 0x0800)

    Non-saturating performance counters are supported. Upon counter value wraps around, a corresponding interrupt can be generated. The performance counters are read to clear, and can be frozen by writing 0 to the ENA.

    Bits Name Type Reset Description
    1 RTC RW 0 1: performance counters are read to clear.
    0 ENA RW 1 1: enable all performance counters. when ENA=0, freeze. when ENA goes to 1 (from 0), reset and enabled. when ENA=1, enabled.




    Performance Counter Control 1 (MICA_COMP_ENG_INFL_REG_PERF_CTL_1: 0x0840)

    Performance Counter Control 1
    MODEx: performance counter mode selection for PERF_CNT_x

    0: event count - number of memory copy (ENGINE_TYPE=0/1) started
    1: event count - number of transaction (ENGINE_TYPE=3) processed
    2: event count - number of transaction (ENGINE_TYPE=3) started
    3: event count - number of GZIP transaction started
    4: event count - number of deflate final blocks started (BTYPE=0/1/2)
    5: event count - number of deflate blocks (BTYPE=0) started
    6: event count - number of deflate blocks (BTYPE=1) started
    7: event count - number of deflate blocks (BTYPE=2) started
    8: event count - number of deflate blocks (BTYPE=3) started
    12: event count - number of cycles that the inflate engine is busy working on the transaction
    13: byte count - number of memory copy bytes (ENGINE_TYPE=0/1) started
    14: byte count - number of input bytes (ENGINE_TYPE=3) started, including all header/trailer
    15: byte count - number of output bytes (ENGINE_TYPE=3) processed

    If there is no in-flight transaction,
    counter(mode=1) = counter(mode=2)

    number of transactions in raw deflate format = counter(mode=2) - counter(mode=3)

    number of deflate blocks (from gzip or raw deflate input) = counter(mode=5) + counter(mode=6) + counter(mode=7) + counter(mode=8);
    number of deflate blocks (from gzip or raw deflate input) may not be same as counter(mode=4), e.g. one transaction may have multiple blocks.

    Bits Name Type Reset Description
    59:56 MODE7 RW 15 Performance counter mode selection for PERF_CNT_7 (same encoding as MODE0).
    51:48 MODE6 RW 14 Performance counter mode selection for PERF_CNT_6 (same encoding as MODE0).
    43:40 MODE5 RW 5 Performance counter mode selection for PERF_CNT_5 (same encoding as MODE0).
    35:32 MODE4 RW 4 Performance counter mode selection for PERF_CNT_4 (same encoding as MODE0).

    3:0 MODE0 RW 0 Performance counter mode selection for PERF_CNT_0 (see description).

    Interrupt vector, write-one-to-clear (MICA_COMP_ENG_INFL_REG_INT_VEC_W1TC: 0x1000)

    Each bit in this register corresponds to a specific interrupt. Hardware sets the bit when the corresponding condition has occurred. Writing a 1 clears the status bit. Bit definitions are provided in the INT_VEC.

    Bits Name Type Reset Description
    9:0 REG_INT_VEC_W1TC W1TC 0 Each bit in this register corresponds to a specific interrupt. Hardware sets the bit when the corresponding condition has occurred. Writing a 1 clears the status bit. Bit definitions are provided in the INT_VEC.




    Interrupt Vector Mask (MICA_COMP_ENG_INFL_REG_INT_VEC_MASK: 0x1040)

    Each bit in this register corresponds to a specific interrupt. When set, the corresponding interrupt will not be dispatched. Bit definitions are provided in the INT_VEC.
    Bits Name Type Reset Description
    9:0 REG_INT_VEC_MASK RW 0 Each bit in this register corresponds to a specific interrupt. When set, the corresponding interrupt will not be dispatched. Bit definitions are provided in the INT_VEC.
