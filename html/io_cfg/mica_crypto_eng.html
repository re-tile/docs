    Token Control and Status (MICA_CRYPTO_ENG_TOKEN_CONTROL_STATUS: 0x0000)

    Provides control and status for the packet processor.
    Bits Name Type Reset Description
    31 HOLD_PROCESSING RW 0 Stops processing packets after the currently loaded token. If this bit is cleared, the number of packets that needed to be processed is continued.
    29:24 PROCESS_N_PACKETS RW 0 A fixed number of packets will be processed A write to this register increments the number of packets that will be processed with the written value. Valid values are 0 (minimum) to 63 (maximum), but the sum of Process N packets and Packets to be processed may not exceed 63.
    23 DEBUG_MODE RW 0 Enables the packet processor debug mode. In this mode, a specific number of packets can be processed while the processing can be stopped after every token.
    22 TIME_OUT_COUNTER_ENABLE RW 0 Enables the time-out counter that generates an error in case of a hang situation. If this bit is not set, the time-out error can never occur.
    17 INTERRUPT_PULSE_OR_LEVEL RW 0 If interrupt output will be a level signal (1) that will be active until it is cleared by the Host, or the interrupt will be pulsed (0) for one clock cycle.
    16 OPTIMAL_CONTEXT_UPDATES RW 0 Pipelines the context updates. This allows the engines to perform context updates and unlocks to be performed after the next packet processing has started.
    15 BUSY RW 0 Packet engine is busy (a context is active).
    14 PROCESSING_HELD_IDLE RW 0 No (part of) tokens are in the packet processor and no context update required.
    13:8 PACKETS_TO_BE_PROCESSED RW 0 Indicates the remaining number of packets to be processed before the processing is held. Valid in debug mode only, see bit [23].
    7 RESULT_CONTEXT RW 0 The context cache contains result context data that needs to be updated.
    6 CONTEXT_FETCH RO 0 The context cache is currently filled.
    5 CONTEXT_CACHE_ACTIVE RO 0 The context cache contains a new context.
    4 TOKEN_READ_ACTIVE RO 0 A token is currently read by the packet processor.
    3 RESULT_TOKEN_AVAILABLE RO 0 A (partial) result token is available in the packet processor.
    2 TOKEN_LOCATION_AVAILABLE RO 0 A new token can be read by the packet processor.
    1:0 ACTIVE_TOKENS RO 0 Number of tokens located in the packet processor, result token not included. (maximum is two).

    TRNG Run Test State (MICA_CRYPTO_ENG_TRNG_RUN_CNT: 0x8080)

    These registers are used to access the 10 counters used to perform a run test and long run test as specified by the AIS-31 standard (tests T3 and T4, ref [11]). They are also used to perform the noise source failure test proposed in section E.5 of that same standard.
    The number of consecutive zeroes and ones (runs) in the data stream shifted into the main LFSR is counted. The run length and bit value is then used to increment a specific bucket counter for these values. After 20000 bits, the bucket counters should be within specified limits for this test to pass. If not, a run_fail interrupt is generated.

    TRNG Poker Test Result (MICA_CRYPTO_ENG_TRNG_POKER_3_0: 0x80c0)

    These registers are used to access the 16 counters used perform a poker test on blocks of 20000 bits (in parallel to the run test). Each of these 16 counters counts the occurrences of one specific 4-bit value in the data stream fed into the main LFSR, with all the counters decremented by one every 64 data bits and reset to their start value every 20000 bits. All counters start at value 1 and are decremented 312 times during the 20000 bit test run.
    Each 8 bits counter holds a 2s complement value and will not overflow past the range 128 +127. At the end of the 20000 bits block the values of the counters with a single 1 bit appended at the least significant bit side are individually squared and then added together the poker test fails (with a poker_fail interrupt) when the resulting sum is outside the range 1288 71749 or one of the counters tries to increment or decrement outside its limit range.
    Note
    s:
    The poker test fails when the 4 bits values of the data stream are distributed too evenly (with 8 counters having incremented 312 times and the others incremented 313 times), this is fully intentional. The minimum mean deviation from the expected value of 312.5 is 4.5.
    Failure at counter overflow is not an official part of the poker test as specified in the AIS-31 standard (ref [11]). It can be shown that the maximum deviation for one counters value from the mean value of 312.5 (without the poker test failing) is 129.5 as this deviation is more than 40% of the mean value it is clearly an indication that something is wrong.
    When the cont_poker bit in the TRNG_TEST register is set to 1, the test is not stopped after 20000 bits the counters keep incrementing and decrementing (the latter every 64 bits) and a poker_fail interrupt is generated when one of the counters tries to increment or decrement outside its limit range.
    Poker test result registers are all read-only writing them accesses TRNG_V_... registers mapped at these same addresses. All counters in these registers are reset when writing a 1 to either the run_fail_ack or the poker_fail_ack bits in the TRNG_INTACK register.

    31:24 POKER_COUNT_3 RO ffh Poker test counter for 4-bits value 3.
    23:16 POKER_COUNT_2 RO ffh Poker test counter for 4-bits value 2.
    15:8 POKER_COUNT_1 RO ffh Poker test counter for 4-bits value 1.
    7:0 POKER_COUNT_0 RO ffh Poker test counter for 4-bits value 0.

    TRNG Test (MICA_CRYPTO_ENG_TRNG_TEST: 0x80e0)

    This register can be used by the Host to perform a number of tests on the TRNG logic:
    Register controlled characterization by connecting the tst_fro_clk_out output to a selected FRO clock output.
    FRO logic connectivity and error event detection checking by feeding known patterns through the FRO delay line and error event detection circuits.
    Direct XOR-ed FRO outputs capture by disabling the main LFSR feedback logic.

    6 TEST_POST_PROC RW 0 1 = Provide direct access to the post-processor for known-answer tests (writing input data to the TRNG_INPUT_... registers).
