This document describes the following groups of registers:

  • SERDES (0x0000-0x00ae) : These registers are intended for diagnostics. The SERDES control registers are accessed through the SERDES_CONFIG registers in mPIPE and TRIO configuration space. Access is not generally required for normal operation of the PHY. The initial values depend on the PHY configuration (XAUI vs. SGMII vs. PCIe vs. StreamIO etc.).

    Register Definitions



    Control register 0 (SERDES_CTRL_0: 0x0000)

    Note: This register can be reprogrammed when the PHY is under reset or when calibration has completed (PMA is ready).
    Bits Name Type Reset Description
    7 AUTO_SHFT RW HW Auto Shift. Defines whether Electrical Idle 1 pattern is automatically shifted in SERDES macro after loading the drive pattern. When set to 1, Electrical Idle I or Drive mode can be entered within a single aTxClkp clock cycle. When set to 0, 23 clock cycles are required to dynamically switch between Electrical Idle I and Drive mode. In general, this bit is always set to 1.
    6 FORCE_RX_DET RW HW Force Receive Detect. Force the result of PCIe receiver detect operation to be always detected. This register can be used on un-reliable result of Rx detect operation. When set to 1, the result of PCIe receiver detect operation is always positive and thus make the PHY non-compliant to PCIe specification. By default, this bit must be set to 0 and is don't care for other protocols.
    5:4 CDR_REF RW HW CDR reference: these 2 bits are used to define the CDR reference PLL mode. By default, these 2 bits must be set to 00b when aRefClk is used for CDR reference clock.
    3 PMA_DRV_MD RW HW PMA Driven Mode. This bit put the CDR PLL in PMA driven mode. Note that PCS driven is always implemented in the PMA control logic and can be used for any protocol. When set to 0, the PCS driven mode is selected for locking the SERDES CDR circuitry and when set to 1, the PMA driven mode is used.
    2 CDR_PLL_DELTA RW HW CDR PLL Delta. This register defines the frequency comparator threshold value to switch from fine grain locking to frequency lock and thus control the aRxFDIp0 input signal of the PMA macro when CDR is configured in PMA driven mode and the equivalent function when the PMA is configured in PCS driven mode. When set to 0, Rx clock and Tx clock must be in 0.4% difference range, 0.8% when set to 1.
    1 SIG_DET_THD RW HW Signal Detect Threshold. This register defines the Schmitt trigger signal detection threshold used to detect Electrical Idle on Rx and thus control the aSigDetSetting input signal of the PMA macro. When set to 0, threshold is 125 mV (+-40%), and when set to 1, threshold is 180 mV (+-33%).
    0 TX_SEL_FB RW HW Tx Select Rx Feedback. This register directly drives the aTxSelRxfb input of the SERDES.
    This register must be set to 0 when aRefclk is used for Tx PLL and to 1 when CDR PLL is used as Tx PLL reference clock.

    7:6 RXIDLE_MSB RW HW These bits are used as the MSB bits of the activity detector logic, enabling to specify that no activity has been detected during up to 61 aTxClkp clock cycles. These bits are the 2 MSB bits whereas the rxidle_max[3:0] field of reg02 represent the LSB part.

When set to zero, the CDR PLL is put in reset and low-power, enabling to save maximum power. When the opposite component send TS1 ordered set to drive the link in recovery, only pipe_rxelecidle signal get de-asserted at PIPE interface and the PHY waits for the controller to change the pipe_powerdown[1:0] signal back to P0, before retraining the CDR PLL (~6 us) and sending received data to the controller. When set to one, the CDR PLL is kept alive in frequency lock mode in P1 state which enable a faster recovery time from P1 state but which also consumes more power (all Rx logic is kept alive and consumes power in P1 state). Note that this register must not be set for application which remove reference clock in P1 mode (generally associated to CLKREQ# signal, Expresscard application and more generally power sensitive application).
