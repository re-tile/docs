This document describes the following groups of registers:

  • Common (0x0000-0x0038) : Each IO device contains a common set of registers used for device discovery and general setup.

  • I2C Master (0x0100-0x0208) : These registers provide control for the I2C master.

    Device Control (I2CM_DEV_CTL: 0x0008)

    This register provides general device control.
    Bits Name Type Reset Description
    3 SDN_ROUTE_ORDER RW 0 When 1, packets sent on the SDN will be routed x-first. When 0, packets will be routed y-first. This setting must match the setting in the Tiles. Devices may have additional interfaces with customized route-order settings used in addition to or instead of this field.
    2 RDN_ROUTE_ORDER RW 1 When 1, packets sent on the RDN will be routed x-first. When 0, packets will be routed y-first. This setting must match the setting in the Tiles. Devices may have additional interfaces with customized route-order settings used in addition to or instead of this field.




    MMIO Info (I2CM_MMIO_INFO: 0x0010)

    This register provides information about how the physical address is interpreted by the IO device. The PA is divided into {CHANNEL,SVC_DOM,IGNORED,REGION,OFFSET}. The values in this register define the size of each of these fields.
    Bits Name Type Reset Description
    40:35 REGION_WIDTH RO 0 Size of the REGION field for this channel. The LSBs of the physical address are interpreted as {REGION,OFFSET}
    34:29 OFFSET_WIDTH RO 16 Size of the OFFSET field for this channel. The LSBs of the physical address are interpreted as {REGION,OFFSET}
    28:22 NUM_SVC_DOM RO 0 Number of service domains associated with this channel.
    21:19 SVC_DOM_WIDTH RO 0 Number of bits of service-domain specifier for this channel. The MSBs of the physical addrress are interpreted as {channel, service-domain}
    18:4 NUM_CH RO 9 Each configuration channel is mapped to a device having its own config registers.
    3:0 CH_WIDTH RO 4 Number of bits of channel specifier for all channels. The MSBs of the physical addrress are interpreted as {channel, service-domain}




    Memory Info (I2CM_MEM_INFO: 0x0018)

    This register provides information about memory setup required for this device.
    Bits Name Type Reset Description
    55:48 NUM_TLB_ENT RO 0 Number of IO TLB entries per ASID.
    47:40 NUM_ASIDS RO 0 Number of ASIDS supported if this device has an IO TLB (otherwise this field is zero).
    35:32 NUM_HFH_TBL RO 0 Number of hash-for-home tables that must be configured for this channel.
    31:0 REQ_PORTS RO 32768 Each bit represents a request (QDN) port on the tile fabric. Bit[15] represents the QDN port used for device configuration and is always set for devices implemented on channel-0. Bit[14] is the first port clockwise from the port used to access configuration space. Bit[13] is the second port in the clockwise direction. Bit[16] is the first port counter-clockwise from the port used to access configuration space. When a bit is set, the device has a QDN port at the associated location. For devices using a nonzero channel, this register may return all zeros.

    Clock Count (I2CM_CLOCK_COUNT: 0x0038)

    Clock Count
    Bits Name Type Reset Description
    15:1 COUNT RO 0 Indicates the number of core clocks that were counted during 1000 device clock periods. Result is accurate to within +/-1 core clock periods.
    0 RUN RW 0 When 1, the counter is running. Cleared by HW once count is complete. When written with a 1, the count sequence is restarted. Counter runs automatically after reset. Software must poll until this bit is zero, then read the CLOCK_COUNT register again to get the final COUNT value.




    Baseline Control (I2CM_BASELINE_CTL: 0x0100)

    Baseline Control
    Bits Name Type Reset Description
    0 ENABLE RW 1 Interface Enable.
    1: to enable the interface
    0: to reset the interface (such as FIFOs and state machines).
    The I2C master will send a start/stop condition to resynchronize the I2C bus when ENABLE is de-asserted.
    ENABLE can be asserted after 120 micro-seconds (100 KHz I2C) or 30 micro-seconds (400 KHz I2C).

    FIFO Count (I2CM_FIFO_COUNT: 0x0110)

    FIFO Count
    Bits Name Type Reset Description
    17:16 BFIFO_COUNT RO 0 n: n active entries in the boot FIFO (max is 2**1). Each entry has 64 bits + controls.
    0: no active entry in the boot FIFO (that is empty).
    11:8 WFIFO_COUNT RO 0 n: n active entries in the write FIFO (max is 2**3). Each entry has 64 bits.
    0: no active entry in the write FIFO (that is empty).
    3:0 RFIFO_COUNT RO 0 n: n active entries in the read FIFO (max is 2**3). Ech entry has 64 bits.
    0: no active entry in the read FIFO (that is empty).

    I2C Write Data (I2CM_WRITE_DATA: 0x0130)

    I2C Write Data
    Bits Name Type Reset Description
    63:0 WDAT RW 0 Specifies the write data. Used by WRITE instruction.
    ADDR_SEL.ADDR_DIS=0: (with address)
    [7:0] contains data in byte address 0, [63:56] contains data in byte address 7. If the starting byte address (ADDRESS register) is not 8-byte aligned, steered data should be put into the WDAT register.

    ADDR_SEL.ADDR_DIS=1: (without address)
    [7:0] contains the first 8-bit of data to be written, [63:56] contains the last 8-bit of data to be written.




    I2C Read Data (I2CM_READ_DATA: 0x0138)

    I2C Read Data
    Bits Name Type Reset Description
    63:0 RDAT RO 0 Contains the read data. Used by READ instruction.
    ADDR_SEL.ADDR_DIS=0: (with address)
    [7:0] contains data in byte address 0, [63:56] contains data in byte address 7.

    ADDR_SEL.ADDR_DIS=1: (without address)
    [7:0] contains the first 8-bit of returned read data, [63:56] contains the last 8-bit of returned read data.

    I2C Byte (I2CM_BYTE: 0x0148)

    I2C Byte
    Bits Name Type Reset Description
    19:0 BYTE RW 0 Specifies the transfer size (in bytes) of the next instruction. Used by READ and WRITE instructions.

    1: 1 byte.
    N: n bytes

    Note: If the ADDR[2:0] are 3'b0, then no restriction on transfer size.
    If the ADDR[2:0] are not 3'b0, then the transfer size can not across the 8 bytes boundary.

    I2C Address Selection (I2CM_ADDR_SEL: 0x0170)

    I2C Address Selection
    Bits Name Type Reset Description
    1 ADDR_DIS RW 0 Address disable
    0: enable transmission of WORD ADDRESS/COMMAND to the slave device after the device address field.
    1: disable transmission of WORD ADDRESS/COMMAND to the slave device after the device address field. The ADDRESS register is not used. This mode should be used if the WORD ADDRESS/COMMAND is not one byte or two bytes.

    A typical I2C write sequence with disabled address:
    write ADDR_SEL.ADDR_DIS = 1
    write BYTE = num_bytes (num_bytes should be no more than the PAGE_SIZE with disabled address)
    write INST = WRITE
    write WDAT for all payloads (except the 7-bit I2C address and the R/W bit)

    A typical I2C read sequence with disabled address:
    write ADDR_SEL.ADDR_DIS = 1
    write BYTE = num_bytes
    write INST = READ
    read RDAT for all payloads (except the 7-bit I2C address and the R/W bit)
    0 ADDR_16_BIT RW 1 1 selects 16-bit i2c address (that is to support EEPROM 32 kb-1 Mb)
    0 selects 8-bit i2c address (that is to support EEPROM 1 kb - 16 kb)




    I2C Prescaler (I2CM_PRESCALER: 0x0178)

    I2C Prescaler
    Bits Name Type Reset Description
    11:0 PRESCALER RW 26ch Prescaler to select the I2C frequency. SCL frequency can be calculated by
    Freq(SCL) = Freq(REF_CLK) / (2* PRESCALE + 9).
    Note:

    If REF_CLK = 125 MHz, the default I2C frequency = Freq(SCL) = 125M / (2*620 + 9) = 100 KHz.




    Glitch Mask (I2CM_GLITCH_MASK: 0x0180)

    Glitch Mask
    Bits Name Type Reset Description
    5:0 CYCLE RW 0 This 6-bit value is used to mask input glitches on SCL and SDA. Any glitches with pulse width (in number of REF_CLK cycles) that is smaller than CYCLE+1 will be removed. For example, with CYCLE=0, any glitches with pulse width of 1 clock cycles or smaller will be removed. Glitches in SCL signal will lock up the I2C controller.




    I2C Write Cycle (I2CM_WRITE_CYCLE: 0x0188)

    I2C Write Cycle
    Bits Name Type Reset Description
    3:0 WC RW 5 I2C EEPROM Write Cycle. Certain I2C EEPROMs require a long write cycle time (tWR). The I2C ROM write cycle applies to a byte write, or a page write. If the master chooses to write one byte at a time, a write cycle has to be applied for each byte. If the master chooses to write a batch of bytes, then the hardware controller tries to honor the page boundary, and does the segmentation. At each page boundary, a write cycle has to be applied. Basically, the write cycle of the shim register is same as the write cycle parameter in the ROM data sheet (e.g. tWR).
    0: 10 REF_CLK cycles.
    1: 1 ms.
    n: n ms (where n = 1 to 8, if REF_CLK = 125 MHz).
    9-15: 8.4 ms.




    I2C No Stop (I2CM_NO_STOP: 0x0190)

    I2C No Stop control
    Bits Name Type Reset Description
    0 CTL RW 0 This controls whether the master generates a STOP or a repeated START condition at the end of a write/read request. This does not apply to a dummy write (the master always generate a repeated START condition at the end of a dummy write).
    1: repeated START condition (no STOP)
    0: STOP condition




    Interrupt vector, write-one-to-clear (I2CM_INT_VEC_W1TC: 0x0200)

    Each bit in this register corresponds to a specific interrupt. Hardware sets the bit when the associated condition has occurred. Writing a 1 clears the status bit.
    Bits Name Type Reset Description
    10 WFE_ERR W1TC 0

    Reserved
    9 RFF_ERR W1TC 0

    Reserved
    8 INST_EXEC W1TC 0 An instruction finished execution
    7 WFIFO_READ W1TC 0 An WFIFO entry was read from the write FIFO
    6 RFIFO_WRITE W1TC 0 An RFIFO entry was written to the read FIFO
    5 WACK_ERR W1TC 0 The external device failed to acknowledge a write
    4 RACK_ERR W1TC 0 The external device failed to acknowledge a read
    3 INST_ERR W1TC 0 The instruction register was written when the controller was busy
    2 WDAT_ERR W1TC 0 The write FIFO was written but it was full
    1 RDAT_ERR W1TC 0 The read FIFO was read but there was no data available
    0 INST_UNKNOWN_ERR W1TC 0 An unknown instruction was executed by the controller
