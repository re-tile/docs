5.51.17 TILE-Gx/TILEPro Loop Pragmas

Both TILE-Gx/TILEPro support the following pragmas that facilitate the compilation of loops:

#pragma unroll n
The user can provide unrolling hints by placing a #pragma unroll n immediately before the loop, where n is a constant specifying the unroll factor. The pragma applies to inner loops. If n is 0 or 1, this tells the compiler not to unroll the loop. Note that the unroll pragma will only be processed if the optimization level is at -O2 or above (it applies at -Os also). The compiler will usually respect the unrolling request, but may make minor adjustment to the unroll factor in certain cases (e.g. make it a power of two to simplify code for cleanup iterations).

If the compiler fails to find a matching loop for a pragma, it emits a warning. This warning can be disabled with the flag -Wno-unmatched-loop-pragmas. Note that loop optimizations (full unrolling, interchange, fusion) may change the code sufficiently to cause a pragma to have no matching loop.

#pragma must_iterate [min] [, [max] [, [multiple]]]
The must_iterate pragma is placed before an inner loop, and it tells the compiler information about the number of times the loop will execute. The compiler may then optimize the loop using these assumptions.

In the pragma, min is the minimum number of iterations, max is the maximum number of iterations, and the number of iterations must be evenly divisible by multiple. All the fields in the pragma are optional; an empty field implies no information on that variable.

If the compiler fails to find a matching loop for a pragma, it emits a warning. This warning can be disabled with the flag -Wno-unmatched-loop-pragmas. Note that loop optimizations (full unrolling, interchange, fusion) may change the code sufficiently to cause a pragma to have no matching loop.
