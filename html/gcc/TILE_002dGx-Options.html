3.17.36 TILE-Gx Options

These ‘-m’ options are supported on the TILE-Gx:

-m32
-m64
Generate code for a 32-bit or 64-bit environment. The 32-bit environment sets int, long, and pointer to 32 bits. The 64-bit environment sets int to 32 bits and long and pointer to 64 bits.
-mbig-endian
Compile code for the processor in big endian mode.
-mbme
Compile using the headers and libraries for Bare Metal Environment (BME).
-mcmodel=small
Generate code for the small model. Distance for direct calls is limited to 500M in either direction. PC-relative addresses are 32 bits. Absolute addresses support the full address range.
-mcmodel=large
Generate code for the large model. There is no limiation on call distance, pc-relative addresses, or absolute addresses.
-mcpu=name
Selects the type of CPU to be targeted. Currently the only supported type is ‘tilegx’.
-mexpand-asm
Expand certain inline assembly into rtl patterns that the compiler can better optimize. It's on by default and is used to support post incrementing memory intrinsics.
-mlittle-endian
Compile code for the processor in little endian mode.
-mnewlib
Compile using the newlib headers and libraries.
-march=name
-momit-leaf-frame-pointer
-mtune=name
These flags are ignored, and are here for x86 compatibility.
-Wunmatched-loop-pragmas
Warn about loop pragmas that do not have matching loops.
