Name

trio-stat — TILE-Gx TRIO I/O Statistics

Synopsis

trio-stat [ { -i | --interval } interval ] [ -r | --rates ] [ -s | --size ] [ -h | --human-readable ] link1 [ link2 | ... | linkN ]

Description

trio-stat prints TRIO (PCIe) I/O statistics, such as bytes and packets sent and received. Note that it is a native tool, and must be run on the Tile Processor itself.

The link must be one or multiple on-chip TRIO interfaces: trio0_mac0, trio0_mac1, trio0_mac2, or trio1_mac0 etc. If no link is specified, trio0_mac0 is assumed.

For example: 

# /usr/sbin/trio-stat trio0_mac0
LINK       Tx Pkt       Tx bits    Rx pkt       Rx bits    Host reads Host bits
-------------------------------------------------------------------------------
trio0_mac0 635788      19934271     88832       9938956       0         0
trio1_mac0 243578       1683427      4423         68837       0         0

The LINK column shows the link names. The Tx Pkt column shows the total number of packets DMA'd from the tile to the host. The Tx bits column shows the total number of bits DMA'd to the host. The Rx pkt column shows the total number of ingress packets, DMA'd from the host. The Rx bits column shows the total number of ingress bits, assuming a fixed size DMA is transferred each time. The default size is 4096 bytes, which can be changed with the --size command line argument. The Host Reads column shows the number of PCIe reads from the host. The Host bits is the number of bits transfers for host reads.

Options

-i | --interval interval

By default, trio-stat prints out statistics once. If you set an interval, trio-stat repeats the statistics every specified number of seconds.

-r | --rates

Print statistics per second instead of counts. Note that the --rates option requires that you set an interval using the --interval option. For example: 

# /usr/sbin/trio-stat trio0_mac0 --interval 10 --rates

LINK       Tx pkt/s        Tx bits/s Rx pkt/s        Rx bits/s Host Reads/s      Host bits/s
--------------------------------------------------------------------------------------------
trio0_mac0        0                0        0                0        0              480
trio0_mac0    35788          1934271    18832           938956        0                0
trio0_mac0    43578           163427     4423             6837        0                0
trio0_mac0    37781          1734712    17321           949568        0                0
trio0_mac0    45783           144277     4532             6773        0                0
...
-s | --size bytes

Specify the size of the DMA transfers from host to Tile. The number of bytes sent from host to Tile is host recorded by the hardware counters, so a bits transferred is computed using a fixed number of bytes per transfer. The default value is 4096 bytes.

-h | --human-readable

Print statistics numbers as human readable format. i.e. K: Kilo (1000), M: Mega (10^6), G: Giga (10^9), T: Tera (10^12) and P: Peta (10^15). 

# /usr/sbin/trio-stat trio0_mac0 trio1_mac0 --interval 10 --rates --human-readable

LINK       Tx pkt/s        Tx bits/s Rx pkt/s        Rx bits/s Host Reads/s      Host bits/s
--------------------------------------------------------------------------------------------
trio0_mac0   35.78K            1.93M   18.83K          938.95K        0                0
trio1_mac0   43.57K          163.42K    4.42K            6.83K        0                0   
...

Notes

The rates statistics are calculated based on the time measured by the system's core reference clock, which is unlikely to be completely accurate; typical oscillators used for this purpose might vary by as much as 300 parts per million (0.03%). For instance, a 28.8 Gbps interface that was in fact completely saturated might be reported to be running at anywhere from 28.791 Gbps to 28.809 Gbps.

When using --interval, or when running trio-stat several times in succession, you may notice that some values seem to reset from zero, while others do not.

  • the values are incremental; each time they're displayed, you see the count of events since the last time the value was displayed. This is because these values are maintained by hardware in clear-on-read registers; whenever trio-stat reads them, they're reset to zero.
