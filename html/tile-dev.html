Name

tile-dev — Utility that determines TILE device to use, based on a specified device name or label, or the default device if no name/label is specified.

Synopsis

tile-dev [options...] [device-name]

Description

Usage

The tile-dev command is used by MDE commands such as tile-monitor and tile-console to determine the actual TILE device to use, based on arguments provided by the user, such as the --dev option.

The tile-dev command can also be invoked by the user to check that the configuration and defaulting of TILE device(s) is correct. (Note that this is purely informational: you do NOT need to invoke tile-dev yourself to use commands such as tile-monitor and tile-console.)

For example, to determine the device name for a TILE device connected to USB port 0: 

  $ tile-dev usb0
  /usb0

To determine the device name for a PCI card installed in the current host: 

  $ tile-dev gxpci0   (for Gx cards)
  /gxpci0

  $ tile-dev pci0     (for TilePro cards)
  /pci0

To determine the device name for the TILE device with the administrator-assigned label tile0: 

  $ tile-dev tile0
  /usb-tile0

Operation

The tile-dev command derives its information from two main sources: actual device names (e.g. /dev/tileusb0), and configuration files (e.g. /etc/tileXYZ.conf) created during the installation of drivers for TILE hardware.

The .conf configuration files provide additional optional information about a TILE device as a set of key/value pairs. In particular, the label key allows the administrator to define one or more convenient "label" aliases for a device.

For example, a TILE device connected via a usb port might be given a label like tile0, allowing the device to be accessed by tile-monitor --dev tile0 instead of tile-monitor --dev usb0.

Note: for USB-based devices, the admin command config-tileusb includes options for setting/renaming the label key for a device. For other device types, the administrator needs to edit the configuration file manually to add/change labels.

The tile-dev command looks at both the actual device names and any labels defined in configuration files in determining the device the corresponds to the provided arguments. It also recognizes when a given name is the hostname or IP address of a network-accessible device.

If a device-name argument is specified, tile-dev searches for a TILE device corresponding to that name or a device whose configuration has a label entry matching that name.

If no device-name argument is specified, but the environment variable TILERA_DEV is set, the value of TILERA_DEV is used as the name of the device to search for, as described above.

If no device-name is specified, and TILERA_DEV is not set, tile-dev determines whether there is an obvious "default" TILE device for the current host. If there is a single TILE configuration file, or exactly one configuration file that matches any specified filter arguments such as --arch tilegx, this is the default device for the current host.

If there are no configuration files on the current host, or there is more than one configuration file, or more than one configuration file matching the specified filters, then there is no default device, and tile-dev exits with a non-zero exit status.

If a device matching the provided arguments is found, tile-dev returns a zero exit status.

Command-Line Options

--help

Optional. Displays usage information and list of command options, then exits.

--quiet

Optional. Does not display error messages, just sets exit status.

--arch

Optional. Filters out configuration files that have an arch key that does not match the specified TILE architecture name (e.g. --arch tilegx).

--key|--optional-key

Optional. Reports the specified key from the configuration file for the selected device, rather than the device name. (Example: tile-dev --key serial reports the console serial device, if any, configured for the default TILE device.)
