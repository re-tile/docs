Name

tile-reboot — Reboot a network-connected TILE board

Synopsis

tile-reboot [option...] { address | --self }

Description

This program reboots the Tile Processor at the specified network address, installing a new bootrom file if specified. If run natively on a Tile Processor, this program can also perform those same functions on that processor. The install of the new bootrom file is skipped if it matches the already-installed bootrom file (as verified by the SHA1 checksum).

The program functions as a wrapper around the ssh/scp tools for file upload and remote command invocations. If the remote board does not allow root logins without a password, you will have to provide your password (or private key) when running. On the remote board, the sbim command is run to display current bootroms and to install a new one if necessary.

If "--bootrom" is specified, you may also request deferring the reboot until later, forcing the image to be burned even if the SHA1 checksum matches the newest boot image on the Tile Processor, or rebooting only if the bootrom was changed.

Generally speaking, tile-monitor is a better choice to use than this tool, unless for some reason you need to statically create bootroms.

Options

--bootrom bootrom-file

Use the specified bootrom to reboot the Tile Processor. See tile-mkboot for how to create a bootrom, or tile-monitor for how to reboot a Tile Processor using a dynamically generated bootrom.

--later

Update the bootrom stored on the board's SROM if necessary, but do not reboot. Only meaningful with the --bootrom option.

--if-changed

Only reboot the Tile Processor if an install of a new bootrom was performed. Only meaningful with a --bootrom option.

--force-install

Install the requested bootrom even if the SHA1 checksum matches. Only meaningful with a --bootrom option.

--self

Act on the local system, instead of acting upon another system on the network. Only valid when running natively.

--batch-mode

Disallow ssh from prompting the user for keys, passwords, etc.

--wait

Wait until rebooting is complete, by waiting for "ping" to fail, and then waiting for "ping" to succeed.

--verbose

Display verbose output.

--help

Display help text.
