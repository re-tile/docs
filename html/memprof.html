Name

memprof — memory bandwidth profiling

Description

The special file /proc/tile/memprof provides an interface for measuring the throughput of a Tile processor's memory controllers. It includes support for starting, stopping, and clearing the memory profile, as well as reading the results. Profile information can only be collected if the system's hypervisor configuration includes a dedicated memprof tile. The default vmlinux-pci.bootrom file includes the memprof tile. For an example of how to add memprof to a hypervisor configuration, see the System Programmer's Guide or examine lib/boot/vmlinux-pci.hvconfig in the MDE installation kit.

Reading the memprof file will generate ASCII-formatted profile results, including the number of cycles profiled, the number of transactions performed on each controller, and a subdivision of those transactions by transaction type (read page hit, read page miss, write page hit, etc). The output can be post-processed by tools like grep in order to extract the statistics of interest.

The following ioctl codes allow the programmer to start, stop, and clear the memory controller profile:

MEMPROF_IOCTL_START

Start collecting profile information.

MEMPROF_IOCTL_STOP

Stop collecting profile information. This does not clear the profile information. Restarting at a later time will continue accumulating into the same profile, but transactions between the stop and start commands will not be counted.

MEMPROF_IOCTL_CLEAR

Clear all profile information, resetting all values to zero. The profile information can be cleared at any time, independent of whether the profiler is currently running.

The start, stop, and clear operations may also be performed by writing the string "start", "stop", or "clear", respectively, followed by a newline, to /proc/tile/memprof. This is convenient for shell scripts, or interactive use.

Errors

If the hypervisor configuration did not include a dedicated memprof tile, attempting to open /proc/tile/memprof will fail and set errno to ENXIO.

Example


#include <asm/memprof.h>

// Open the memprof file.
int memprof_fd = open("/proc/tile/memprof", O_RDWR);

// Clear the current profile and make sure profiler is running.
ioctl(memprof_fd, MEMPROF_IOCTL_CLEAR);
ioctl(memprof_fd, MEMPROF_IOCTL_START);

// Do work here...

// Stop the profile and read the results.
ioctl(memprof_fd, MEMPROF_IOCTL_STOP);
system("cat /proc/tile/memprof");
